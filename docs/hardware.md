## ADS1115 — 아날로그 입력 (익스프레션/볼륨 페달)

ADC 모듈. 페달의 아날로그 전압을 읽어 파라미터로 변환.

- ✅ **I2C**: bus 1, 주소 `0x49` (ADDR 핀을 VDD에 묶은 주소)
- ✅ **VDD**: 3.3V (Pi에서 공급) — **입력 핀 절대최대 = VDD+0.3 ≈ 3.6V. 5V 입력 금지(칩 손상).**
- ✅ 드라이버: [`hardwares/ADS1115.py`](../hardwares/ADS1115.py) (single-shot, 채널 전환식)
- 현재 코드 세팅 ([`volumepedal.py`](../volumepedal.py)):
  - ✅ Gain `±4.096V`, Data rate `128 SPS`, single-shot 모드
  - ch0 → 볼륨, ch1 → 익스프레션

### 게인(PGA)에 관한 핵심
- **게인 설정 = 측정 범위(스케일)일 뿐, 입력 보호와 무관.** 입력 핀 안전 한계는 오직 VDD가 정한다.
- 게인 ±2.048V면 2.048V 넘는 입력이 전부 32767로 **클리핑**됨.
- ✅ **게인 `±4.096V`로 변경 완료** ([`volumepedal.py`](../volumepedal.py)). 페달 와이퍼 상단(~2.36V)이 ±2.048V에선 잘리던 걸 해결.
  - ⚠️ 게인 올리면 같은 전압이 카운트 절반으로 읽힘 → `map_value` 캘리브레이션 상수도 절반(1638/29490 → 819/14745)으로 임시 스케일해둠. **현장에서 페달로 재캘리브레이션 필요.**

### ✅ 검증 로그 — 연결 안 된 상태 진단 (고임피던스 아티팩트 확정)
연결 안 된 상태에서 게인만 바꿔 읽어본 결과:

| 게인 | ch0 | ch1 |
|---|---|---|
| ±2.048V | 32767 (레일/클리핑) | ~29500 = 1.84V |
| ±4.096V | ~23530 = **2.94V** | ~18640 = **2.33V** |

- **같은 물리 노드인데 게인 바꾸니 전압이 변함** (ch1: 1.84→2.33V) = ADS 입력 임피던스가 ~800k 풀업을 분압 중이라는 증거. 깨끗한 저임피던스 소스면 게인과 무관하게 같은 값이 나와야 함.
- → **연결 안 됐을 때 읽히는 값은 신뢰할 DC 측정이 아님.** "페달 꽂혔나" 감지용으로 쓰면 안 됨.
- ⚠️ **ch0 vs ch1 비대칭** = 유효 풀업이 ~3.4배 차이. 저항 공차(±13%)로는 설명 불가 → **두 채널 회로가 실제로 다름**(납땜/단선/풀업값 차이 의심). **현장에서 페달 꽂고 양 채널 힐~토 스윕 비교로 확진할 일.**

> **참고: 풀업 역산 근거 (TI 데이터시트 SBAS444E, VDD=3.3V, DR=8SPS)**
> ADS1115 입력 임피던스 — 차동 / 공통모드:
> ±4.096V = 15M / 6M, ±2.048V = 4.9M / 6M (그 외 ±6.144=22M/10M, ±1.024=2.4M/3M, ±0.512·0.256=710k/100M)
> - 공통모드는 두 게인에서 6M로 동일한데 읽힘값은 게인 따라 변함 → 로딩을 지배하는 건 **차동 임피던스**(단일엔드는 AINN=내부GND라 핀↔GND 경로).
> - 역산 `R_pu = R_adc × (3.3−V)/V`. 차동값 대입: ch0 ~1.8M, ch1 ~3.9~6.2M.
> - ⚠️ **데이터시트는 8SPS, 우리는 860SPS.** 스위치드-캡 임피던스는 빠를수록 낮아져 실제 유효값은 표보다 작음 → 역산 풀업도 더 작게 봐야 함(ch0가 ~1M로 내려와 실측과 부합).
> - **변하지 않는 결론: ch1 풀업 ≈ ch0의 3.4배(가정 무관) → ch0 정상(~1M), ch1 비정상 = 납땜 의심.**

### 데이터레이트 / 노이즈
- ✅ **860SPS → 128SPS로 낮춤** ([`volumepedal.py`](../volumepedal.py)). 폴링 루프가 50ms(20Hz)라 빠른 변환 불필요. 느린 변환 = 내부 평균효과로 노이즈↓ + 입력 임피던스↑로 고임피던스 로딩 완화.

## 익스프레션 페달 입력 (TRS 잭 + ADS1115)

### ✅ 현재 배선 (TRS, 25K 포텐셔미터 전압분배 모드)
```
TRS Tip    ── [10kΩ 직렬저항] ── 3.3V (Pi 전원)
TRS Sleeve ── GND
TRS Ring   ── ADS1115 아날로그 입력 (= 포트 와이퍼)
```
- 페달 포트가 분배기로 동작 → 와이퍼(Ring)가 0V ~ 상단전압을 스윕.
- 10k 직렬저항 때문에 상단 = `3.3 × 25/(25+10) ≈ 2.357V` (3.3V 풀범위의 ~71%만 사용. MIDI 7비트엔 충분).

### ✅ 왜 10kΩ 직렬저항이 있는가 — 핫플러그 셧다운 사후분석
**증상:** TRS 잭에 플러그를 꽂거나 뺄 때 **시스템 전체가 꺼짐(파이 리셋).**

**원인:** 잭 Tip에 파이의 raw 3.3V 전원이 직결돼 있었음. 플러그를 꽂고 뺄 때 금속 배럴이 잭 접점 위를 미끄러지며(scrub), **Tip(3.3V)이 GND와 순간 단락(dead short)** → 3.3V 레일이 무너짐 → 이 레일은 파이 내부도 같이 먹으므로 파이가 브라운아웃·리셋.
- 케이블 종류 무관(TRS여도 발생). 슬라이딩 스크럽은 케이블과 상관없음.
- TS 케이블은 별개로 Ring↔Sleeve를 상시 단락시켜 ADC가 0V로 읽힘 (측정 불가). 하지만 "꺼짐"의 원인은 어디까지나 전원 단락.

**해결:** ✅ **3.3V → Tip 라인에 직렬저항 삽입.** 단락 나도 전류가 `3.3V/R`로 제한돼 파이가 영향 없음.
- 실제 적용값 = **10kΩ** → 단락 전류 0.33mA (매우 안전. 1k였으면 3.3mA, 그것도 충분).
- 트레이드오프: 상단 전압이 2.357V로 낮아져 해상도 약 30% 손실 + 와이퍼 소스 임피던스 ↑(지터에 약간 더 취약). 실사용엔 문제 없음.

### ⚠️ 주의 / 미적용
- **게인 ±4.096V로 올리기** (위 ADS 섹션 TODO). 상단 2.357V가 현재 ±2.048V에선 살짝 잘림.
- 캘리브레이션 하드코딩됨(`map_value` in_min=1638, in_max=29490). 페달 개체차·저항공차 흡수하려면 힐~토 실측 스윕으로 min/max 학습하는 캘리브레이션 루틴 필요(미구현).
- 런타임 지터 억제(EMA/데드밴드) 미적용.

### 참고: 페달 모드별 특성 (스펙)
- 25K TRS/RTS = 전압분배 모드 (현재 사용). RTS는 Tip/Ring 극성 반대.
- 10K TS = 가변저항(rheostat) 모드. **이 시스템 배선(800K~10k 풀업)으론 부적합** — 풀업이 페달저항을 압도해 거의 안 변함. 쓰려면 하드웨어 변경 필요.

## 전원 공급 — 언더볼티지 스터터 사후분석 (2026-07)

Pi 5 (`2712`, 4코어 2.4GHz), 오디오 = pisound HAT. 5V 레일에서 Pi + 모니터 + 오디오카드 + NVMe 동시 구동(고부하).

### 증상
- xrun으로 **안 잡히는** 미세 오디오 스터터. 클로드 코드 실행/CPU 부하 시 심화. 레벨미터 켜져 있을 때 더 도드라짐(부하 얹혀 먼저 드러난 것뿐, 레벨미터 자체는 무죄).

### 원인 (진범 = 전원)
- 기존 **무명 PSU 모듈**이 5V 레일을 Pi 끝단 기준 **4.80V**밖에 못 줌 (거의 무부하에서도 언더볼티지 경고선 ~4.8V에 걸침).
- 부하 스파이크(클로드 등) → 전류↑ → 레일 sag → **~4.63V 밑 → 펌웨어가 SoC를 1.0GHz로 강제 클램프.** cpufreq(governor) **아래 계층**이라 xrun으로 안 잡히고, governor·JACK 버퍼·RT커널 **전부 무관**.
- ❌ 헛다리: `performance` governor는 오히려 전류만 더 먹어 언더볼티지 악화(`0x50000`→`0x50005`로 현재비트 켜짐 확인).

### 해결 ✅
- **외장 EU USB-C 브릭(정품급)으로 전환** → Pi USB-C 직결(PD 협상 = 풀 5A 예산).
- 결과: `EXT5V_V` **5.07V** 안정, `VDD_CORE` 0.72V→0.91V(클램프 해제), `get_throttled` 현재비트 0.

### 진단 명령 (재발 시)
```
vcgencmd get_throttled          # 하위비트0=UV now,2=throttle now / 상위16,18=이력. 0x0 목표
vcgencmd pmic_read_adc | grep EXT5V   # 실제 5V 레일 실측. 부하 걸고 ≥5.0V 유지 확인
vcgencmd measure_clock arm      # 부하 시 2.4GHz 못 오르면 클램프 의심
```
- **핵심**: 스터터의 원인은 "PD 없음"이 아니라 **전압 낮음**. GPIO 5V 급전도 5.1V·저저항이면 OK(단 PD 없으면 USB-A 포트 600mA 제한 → 필요시 `usb_max_current_enable=1`).

### 재설계 (진행 중) — 브릭은 악기 밖에
- **왜 브릭을 박스에 안 넣나**: SMPS = 변압기 코일 자기장 + 광대역 스위칭 EMI. 고임피던스 기타 프론트엔드 옆에 두면 노이즈 유입. 접지핀 하나 얻자고 코일을 악기에 넣는 건 나쁜 설계.
- **전원 인입**: 브릭 일체형 케이블 → **그로밋/글랜드 직결** → Pi USB-C (커넥터 junction 0 = sag 최소). 그로밋 뒤에서 케이블 고정(스트레인릴리프), Pi 커넥터에 장력 금지.
- **접지(기능접지 FE)**: 2핀 브릭이라 접지 없음 → 험(터치험: 기타 줄 만지면 험 변함) 발생. 패널에 **바나나잭/러그** → 내부에서 Pi/오디오 GND에 본딩 → 필요시 얇은 어스선으로 벽 PE. **탈착식**(접지 앰프로 나갈 땐 분리해 그라운드 루프 회피). 전력 안 흐르는 도체라 방사 없음 → PSU와 달리 박스 안/밖 무관.
- **개념 구분**: 그라운드 네트(악기→Pi GND→PE)는 **이어지는 게 정상**(신호 기준, 단일점 접지). "이격" 대상은 **220V L/N 라이브 도체**뿐(고임피던스 신호에 50Hz 용량성 결합 방지). L/N은 꼬아서 짧게, 신호선과 직각 교차.

### ⚠️ 물리 (미완)
- 기존 IEC+스위치+퓨즈 콤보 모듈 자리(**나사폭 40mm, 컷아웃 25×50mm 세로 직사각**) → **블랭킹 플레이트**로 메우고 USB-C + 바나나 접지단자 배치 예정. 기존 40mm 나사홀 재사용.
- 설치 후 **풀부하(모니터+NVMe+오디오) EXT5V ≥5.0V, throttled 0x0 재검증** 필수.
