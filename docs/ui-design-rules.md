# UI 디자인 룰 — Qt 이주 (7" DSI 800×480)

> Pedal Prototype(Claude Design 시안)을 실 하드웨어에 맞춰 구현하기 위한 룰.
> **값은 전부 튜너블** — 온디바이스 테스트 후 조정 전제(특히 §2 헤더 크기).
> 시안 출처: claude.ai/design `Pedal Prototype.dc.html`. 관련: `docs/qt-roadmap.md`.

## 1. 디스플레이 / 물리
- 800×480, **5:3, ~133 PPI**, ≈152×91mm. → **1mm ≈ 5.25px.**
- **풀스크린, 베젤·페이지 크롬 없음** (시안의 디바이스 프레임·스크린 아래 하드웨어 행은 발표용 장식. 실 앱 = 800×480 스크린 하나). 가로 고정.
- 엣지 안전마진 ≥ 12px (DSI 터치가 가장자리서 부정확).

## 2. 두 사용 맥락 = 두 가시성 Tier (핵심)
| | **Tier 1 · 라이브/바닥** | **Tier 2 · 연습/근거리** |
|---|---|---|
| 거리 | ~1.5m (서서 내려다봄) | ~0.5m |
| 입력 | **풋스위치만** | 터치(노드 탭·노브 드래그) |
| 화면 역할 | **글랜스**: 페달보드명·스냅샷 | 전체 인터랙션 |
| 가시성 | 크게 | **최소 확보로 충분** |

**Tier 1 헤더 사이징** (1.5m, 글랜스 시야각 ~0.7° → 글리프 ≈18mm ≈95px):
- 페달보드명: 글리프 높이 **~80–95px** (VT323 font-size ~100–120, 온디바이스 튠).
- 스냅샷(번호+이름): **~40–56px**.
- 상단 **~120px 밴드**를 글랜스 헤더로 전용 (600→480서 줄어든 120px을 여기 투자). 테스트 후 축소 가능.

**Tier 2**(근거리): 모든 그래프·포커스·노브·메뉴. 라벨·이름은 가시성 최소 확보면 됨.

## 3. 타입 스케일 (VT323; 한글은 mago/Galmuri 폴백)
- Tier 1: 페달보드명 80–95px · 스냅샷 40–56px (위 §2).
- Tier 2: 디스플레이 리드아웃(튜너·BPM·탭) 100–140 / 값·섹션 20–24 / 라벨 16–20 / **보조·노브이름 14–16**.
- **<14px는 금지** 영역(근거리여도 하한).

## 4. 터치 / 입력
- **최소 타깃 48px(≈9mm)**, 1차 액션 56–64px. **거리 무관**(손가락 크기) — 연습 맥락이어도 유지.
- 타깃 간격 ≥ 12px. ⚠️ 시안의 저장/다른이름/불러오기·메뉴 버튼이 ~34px라 **터치 미달 → 48px로 키울 것**.
- 제스처: **탭**(선택/토글/버튼), **세로 드래그**(노브). hover·롱프레스·멀티터치 **안 씀**(정전식 단일터치, hover 없음).
- 노브: 세로 드래그 + **값 리드아웃 필수**, 히트영역 ≥56px. 라이브 비사용이라 미세조절 난이도 허용.

## 5. 풋스위치 / 입력 시뮬 (결정)
- 스크린 **안**의 풋스위치 상태 스트립(LED+라벨) = **진짜 UI**(할당·상태 표시) → 구현.
- 시안 스크린 **아래** 하드웨어 풋스위치 행 = 목업 장식 → **구현 안 함**.
- Windows 개발 시 풋스위치 입력 = **키보드 shim(Z/X/C/V → FS0..3) → `view.footswitchKey` → `FakeController` → `presenter.handle_footswitch_event`**. 위젯 아님(지울 것 없음). Pi에선 물리 스위치가 구동.
- `FakeController`는 **no-op 유지**(하드웨어 seam 안 건드림). [[ui-migration-direction]] Step B 결정과 일치.

## 6. 컬러 토큰 (시안 추출, 시맨틱 고정)
- 배경: screen `#0e1118` / panel `#141925`·`#161b26` / elevated `#1d2433`
- 보더: `#2c3648` / `#232b3a`
- 시맨틱: green `#5fd0a0`(active·clean·확인) / orange `#d99a4e`(drive·nav·tap) / red `#e8694a`·`#e6402e`(feedback·bypass) / blue `#3b6fe0`(engaged) / purple `#b58af0`(tuner·edit)
- 텍스트: `#e8edf4`·`#fff`(주) / `#9aa3b2`(보조) / `#5a6270`·`#7e8694`(흐림)

## 7. 레이아웃 / 리플로우 (시안 600 → 실 480)
- **OVERVIEW**: 글랜스 헤더 ~120 + 라우팅 그래프 ~170–200 + 범례/풋스위치 스트립 ~120. **무스크롤.**
- **FOCUS**: 헤더 + 이펙터 카드 + IN/OUT 라우팅 + 노브(flex) + 힌트. 노브영역이 120px 손실 흡수.
- 1차 화면(OVERVIEW/FOCUS)은 무스크롤, 리스트성(브라우저 등)만 스크롤.

## 8. 기타 원칙
- hover 의존 금지(상태 항상 보이게). 글랜스 피드백 = 토스트/LED(시안 유지).
- 얇은 1px·저대비 회피(무대 가독성). 다크+밝은 액센트 유지.
- 폰트: VT323(+Pixelify Sans) **OFL 번들** → `resources/fonts`, 오프라인 Pi 대응. 한글은 기존 mago/Galmuri 폴백.

## 9. 비범위(시안엔 있으나 phase-1 나중)
튜너·탭템포 엔진, 온스크린 키보드, 로드 브라우저, 3층위(BOARD/SNAPSHOT/이펙터) 저장·불러오기 풀구현. phase-1 = OVERVIEW(그래프+스트립) + FOCUS(노브/바이패스) + 스냅샷·BPM 표시 + (키보드 shim)풋스위치 동작.
