# 인코더 레일 인디케이터 — 구현 노트

출처: Claude Design *"Encoder Rail Indicator"*. 워크플로우 검토에서 "어느 인코더를 언제
쓰는지 / 스크롤이 필요한 순간이 헷갈린다"는 발견 → **화면 좌측 3px 엣지 레일**로 상시 표시.

## 개념
- 인코더가 화면 좌측에 세로로 스택(ENC0 위 · ENC1 아래) → **좌측 엣지 3px를 위/아래로 갈라**
  각 인코더의 현재 관여를 비춘다. 레일이 물리 노브를 되가리킨다(매핑 학습 불필요).
- **정적 표시만** — 상태 바뀔 때만 다시 그림, 애니메이션 없음 → 디스플레이 갱신 부담 0.
- **순수 흑백** — 색은 RGB LED 몫(도메인/위험/back). 레일은 색이 못 하는 것만: "지금 살아있는
  손 + 목록 위치". 위험색(red)은 레일에 안 싣는다.
- 한 존 = 4상태 중 하나:
  - `off` — 죽은 손. 검정.
  - `idle` — 쓸 수 있으나 주역 아님. 가로줄 1:2 디더(3px마다 1px on).
  - `(pos,total)` **list** — 목록 주역. 디더 트랙 + 밝은 솔리드 썸이 스크롤 위치. 썸 최소 8px.
  - `solid` — 값 조절/단일 동작 전담. 꽉 참.

## 구현 (app.py)
- `_rail(d, z0, z1)` + `_rail_zone(d, z, zy, zh)` — 좌측 3px 렌더러. 존: ENC0=y2..59, ENC1=y68..125.
- `rails(st)` — 상태→`(z0, z1)` 분류기. **필요 데이터가 전부 `AppState`에 이미 있어 계산은 공짜.**
- `render(st)` 래퍼가 `_frame(st)`(기존 dispatch) 위에 레일을 오버레이 → **한 곳에서 전 상태 커버**.

## 상태별 레일 매핑 & 비용
| 상태 | ENC0 | ENC1 | 재배치 비용 |
|------|------|------|------------|
| CHAIN | list(node) | idle / **solid**(lock 시) | IN·노드스트립·<SYS·노브col0 +3px |
| GLANCE | list(board) | list(snap) | **dots 제거**(레일이 위치 표시), 내용 +2px |
| SLOT MENU | list(menu) | idle | 좌측 정보패널 x0→x4 |
| PICKER | **off**(죽음) | list(cat/fx) | 카테고리 스트립 x0→x5, **carets 제거** |
| MOVE | **solid** | off | (내용 이미 x4~, 무비용) |
| SUB(board) | list | idle | 무비용 |
| SUB(snap) | idle | list | 무비용 |
| NAMING(board) | list(term) | idle | 무비용 |
| NAMING(snap) | off | **solid** | 무비용 (Q3: day 자동→E1 단독이 레일에 그대로) |
| CONFIRM | solid/off (커밋손) | off/solid | 무비용 (위험색은 LED) |
| SYSTEM | list(item) | idle | 무비용 |
| TUNER | **solid** | off | 무비용 (E0 hold 나감, back 규칙 일치) |

**비용 요약 (사용자 관찰이 맞았음):**
- **무비용(9개)** — 내용이 이미 x≥4에서 시작. 레일은 st만 읽어 그림.
- **재배치 필요(3개)** — 좌측 x0에 목록/패널이 앵커된 곳: `_chain` 노드스트립, `_menu` 정보패널,
  `_pick` 카테고리 스트립. 각 +3~5px 시프트.
- **디덥(2개)** — 기존 스크롤 표현이 레일과 중복: GLANCE 하단 `dots()`(원형 마커), PICKER `_caret`
  화살표 → 제거하고 레일 썸을 단일 스크롤 진실로. (`_caret` 함수도 삭제.)

## 확정 사항 (design §05)
- 폭 3px 고정 · 정적만 · 순수 흑백(디더 1:2) · 인코딩 A(존재+스크롤 썸) 단독.
- 콘텐츠 여백 재조정: 위 3개 화면 완료.
- 썸 최소 높이: `max(8, zh//total)`로 하한 8px 확보(긴 목록에서 뭉개짐 방지).
- **[사용자] 존 분할 = 콘텐츠 분할선 추종(`_rail_split(st)`).** design은 1:1(물리 인코더
  정렬)을 제안했지만, 0계층(체인)은 노드스트립:노브 ≈ 3:7이라 1:1 레일이 안 맞았음.
  → 체인/move=y40(갭이 hline에 정렬, ~3:7) · glance=y56(pb/snap 분할) · 모달=y64(~1:1,
  인코더가 상/하 콘텐츠로 안 갈리므로 중립). 각 화면에서 레일이 자기 콘텐츠 옆에 붙어
  "레일이 노브를 되가리킨다" 원칙이 강화됨. (물리-정렬 → 콘텐츠-정렬로 트레이드.)

## 남은 검토
- **패널 실측** — 실제 3px 가로줄(1:2)이 목표 명도로 읽히는지 하드웨어 확인(디더 간격 튜닝 여지).
- **TUNER ↔ Q1** — 레일은 E0-solid(=E0 hold 나감)로 그리지만, 현재 코드는 "아무 press 이탈"
  유지(Q1). 레일은 힌트라 상충 아님. 튜너 이탈을 E0-hold 전용으로 좁힐지는 Q1 재검 시 함께.
- **네이티브 픽셀 시안** — 원하면 128px 그리드로 픽셀-정확 재확인.
